实施计划：分 3 个阶段

由基础设施 → 核心体验 → 视觉打磨，每阶段可独立验证

Phase 1 · 基础设施 优先

6. 全局交互规范

创建统一的 Skeleton 骨架屏组件替代所有 inline spinner；统一 ErrorBoundary 错误处理；标准化空状态组件 EmptyState

5. 侧边栏交互

快捷键 Ctrl+B 折叠、面包屑导航、展开/收起动画优化、活跃状态指示器增强

8. 视觉细节打磨

统一圆角（rounded-xl 卡片）、阴影层级（sm/md/lg）、过渡动画（duration-200）、色彩语义（success/warning/destructive）

Phase 2 · 核心体验

2. 文章编辑器升级

媒体库按钮 → 弹窗选图插入 Markdown、Ctrl+S 快捷键、30s 自动保存到 localStorage、字数/阅读时间实时显示

3. 文章列表改进

前端分页（每页 10 条）、状态筛选标签（全部/已发布/草稿）、按日期/标题排序

1. 仪表盘增强

用 Skeleton 替代 Spinner、图表区域增加卡片阴影、统计数据增加环比变化指示、最近文章改为列表带缩略图

7. 设置页扩展

新增站点信息表单（名称、描述、Logo URL、社交链接），修改后更新 siteConfig 和 .env.local

Phase 3 · 增强功能

4. 媒体管理器完善

拖拽上传区（Drag & Drop）、批量选择删除、缩略图网格优化、复制反馈动画

技术方案

所有组件基于 shadcn/ui，不引入额外设计系统

将创建的新组件

  • Skeleton — 内容占位骨架屏（已安装）
  • EmptyState — 统一空状态（图标+标题+描述+CTA）
  • MediaPicker — 媒体选择弹窗
  • Pagination — 分页组件
  • Breadcrumb — 面包屑导航

将增强的现有组件

  • PostEditor — 媒体库集成 + 自动保存
  • AdminSidebar — 快捷键 + 面包屑
  • Dashboard — 骨架屏 + 视觉增强
  • MediaManager — 拖拽上传 + 批量删除
  • Settings — 站点配置表单
